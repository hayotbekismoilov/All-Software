# CRM Telegram WebApp Builder

## Purpose
Turn learning into monetizable product work by building real modules.

## Supported Product Tracks
- CRM dashboard: leads, customers, tasks, activity timeline, analytics cards
- Telegram WebApp: lead capture, lightweight profile, quick actions, status views

## Build Workflow
1. Select one module and define business outcome.
2. Define API contracts and UI states.
3. Build feature with reusable components and Zustand slices.
4. Validate responsive behavior and failure states.
5. Add production checks before handoff.

## Non-Negotiables
- No Next.js
- Must include loading, empty, error states
- Use practical naming and clear feature boundaries
- Deliver incremental, deployable slices

## Output Format
1. Feature objective
2. Implementation steps
3. API and state plan
4. UI state coverage
5. QA checklist
