# grafana-app-sdk CLI Reference

## Installation

```bash
# Latest version
go install github.com/grafana/grafana-app-sdk/cmd/grafana-app-sdk@latest

# Specific version
go install github.com/grafana/grafana-app-sdk/cmd/grafana-app-sdk@v0.x.y

# Verify
grafana-app-sdk version
```

## Project Commands

### `project init`

Initialize a new app project.

```bash
grafana-app-sdk project init <module-path>
```

**Arguments:**
- `<module-path>` — Go module path (required), e.g. `github.com/example/my-app`

**What it creates:**
- `kinds/` — empty kinds directory with an initialized `cue.mod`
- `pkg/` — Go package skeleton
- `go.mod` / `go.sum` — Go module files
- `Makefile` — build targets including `generate`
- `local/` — local development config

---

### `project kind add`

Add a new kind scaffold to the `kinds/` directory.

```bash
grafana-app-sdk project kind add <KindName> [flags]
```

**Arguments:**
- `<KindName>` — PascalCase kind name (required), e.g. `IssueTracker`

**Flags:**
| Flag | Description |
|------|-------------|
| `--overwrite` | Overwrite existing kind files (use when re-scaffolding) |

**What it creates:**
- `kinds/<kindname>.cue` — common kind metadata
- `kinds/<kindname>_<version>.cue` — <version> schema scaffold with comments

**Notes:**
- Always use `--overwrite` to regenerate when the scaffold changes in newer SDK versions
- The generated file is heavily commented — read the comments carefully
- It will always generate only a single version, the `preferredVersion` in `manifest.cue`. To add additional versions, see the `cue-kind-definition` skill

---

### `project component add`

Add a component scaffold to an existing project.

```bash
grafana-app-sdk project component add <component>
```

**Components:**

| Component | Description |
|-----------|-------------|
| `operator` | Adds operator (standalone binary) scaffolding: reconciler stubs, admission handler stubs, `cmd/operator/main.go` |
| `frontend` | Scaffolds a Grafana plugin frontend using `create-plugin` (requires `yarn`) |

**`operator` — what it creates:**
```
cmd/
└── operator/
    └── main.go          # operator entrypoint
pkg/
├── app/
│    └── app.go           # app.App implementation stub with TODOs
└── watchers/
    └── watcher_<kind>.go # <kind> watcher with stubbed methods
```

**`frontend` — what it creates:**
```
plugin/
├── src/
│   ├── module.ts
│   └── pages/
├── package.json
└── plugin.json
```

## Generate Command

### `generate`

Generate Go types, TypeScript types, AppManifest, and CRD files from CUE kind definitions.

```bash
grafana-app-sdk generate [flags]
```

**Flags:**
| Flag | Description |
|------|-------------|
| `--source` | Path to CUE definitions (default: `kinds/`) |
| `--config` | The CUE selector of the config object (see `CUE Config Object` section) |

**What it generates:**
| Output | Location | Controlled by |
|--------|----------|---------------|
| Go type structs | `pkg/generated/<group>/<version>/types_gen.go` | `codegen.go.enabled` |
| Go client | `pkg/generated/<group>/<version>/zz_generated_*.go` | `codegen.go.enabled` |
| TypeScript types | configured output dir | `codegen.ts.enabled` |
| AppManifest (Go) | `pkg/app/appmanifest_gen.go` | always |
| AppManifest (JSON) | `appmanifest.json` | always |
| CRD (JSON or YAML) | `config/crd/` | CUE config output setting |

**⚠ Generated files must not be edited manually** — they are overwritten on every generate run.

## YAML Bundle Command

### `generate yaml-bundle`

Generate a Kubernetes YAML bundle for deployment.

```bash
grafana-app-sdk generate yaml-bundle [flags]
```

Produces CRDs, webhook configurations, and RBAC manifests ready for `kubectl apply`.

---

## Local Development

### `local`

Commands for running the app locally with a mocked API server.

```bash
# Start the local development environment
grafana-app-sdk local/up
```

Starts a local Kubernetes-compatible API server for development without a full cluster.

## Common Makefile Targets

Projects generated by `project init` include a Makefile with these targets:

| Target | Description |
|--------|-------------|
| `make generate` | Run `grafana-app-sdk generate` |
| `make build` | Build the operator binary and/or frontend plugin |
| `make test` | Run tests |
| `make lint` | Run Go linter |
| `make deps` | Vendor in any missing go dependencies |
| `make local/up` | Create a local k3d development environment with Tilt |
| `make local/push_operator` | Pust the built operator image to the local development enviroment k3d cluster |
| `make local/deploy_plugin` | Deploy the latest built plugin to the local development environment's grafana |
| `make local/down` | Take down the Tilt-deployed resources |
| `make local/clean` | Delete the local development environment k3d cluster |

## CUE Config Object

Control generation behavior via a CUE config object in your kinds. This is typically in the top-level kinds file:

```cue
// Codegen output configuration
config: {
    codegen: {
        // goModule allows explicitly setting the module path.
        // If empty the module path is read from go.mod
        // Default: ""
        goModule: ""
        // goModGenPath anchors generated Go output relative to the module root, instead of execution directory
        // Default: ""
        goModGenPath: ""
        // goGenPath sets the output directory for generated Go types.
        // Default: "pkg/generated/"
        goGenPath: "pkg/generated/"
        // tsGenPath sets the output directory for generated TypeScript types
        // Default: "plugin/src/generated/"
        tsGenPath: "plugin/src/generated/"
        // enableK8sPostProcessing runs the Kubernetes code generators after writing files.
        // This option is unnecessary when using the k8s.apiserver's DefaultAppInstaller
        // Default: false
        enableK8sPostProcessing: false
        // enableOperatorStatusGeneration toggles standard status information to be joined onto kinds' schema's status:
        // {
		//  #OperatorState: {
		// 	    // lastEvaluation is the ResourceVersion last evaluated
		// 	    lastEvaluation: string
		// 	    // state describes the state of the lastEvaluation.
		// 	    // It is limited to three possible states for machine evaluation.
		// 	    state: "success" | "in_progress" | "failed"
		// 	    // descriptiveState is an optional more descriptive state field which has no requirements on format
		// 	    descriptiveState?: string
		// 	    // details contains any extra information that is operator-specific
		// 	    details?: {
		// 		    [string]: _
		// 	    }
		//  }
		//  // operatorStates is a map of operator ID to operator state evaluations.
		//  // Any operator which consumes this kind SHOULD add its state evaluation information to this field.
		//  operatorStates?: {
		// 	    [string]: #OperatorState
		//  }
		//  // additionalFields is reserved for future use
		//  additionalFields?: {
		// 	    [string]: _
		//  }
	    // }
        // Default: true
        enableOperatorStatusGeneration: true
    }
    definitions: {
        // genManifest controls whether manifest files are written
        // Default: true
        genManifest: true
        // genCRDs controls whether CRD files are written
        // Default: true
        genCRDs: true
        // path sets where definition files are written
        // Default: "definitions"
        path: "definitions"
        // encoding determines JSON or YAML serialization of CRDs and manifest, allowed values are "json" and "yaml"
        // Default: "json"
        encoding: "json"
        // manifestVersion lets you pin the manifest schema format (legacy v1alpha1 vs default v1alpha2).
        // The v1alpha1 manifest uses CRD-compatible schemas for kinds, the v1alpha2 version contains more information but requires transformation to make kind schemas CRD-compatible.
        // Default: "v1alpha2"
        manifestVersion: "v1alpha2"
    }
}
```

## Troubleshooting

### `grafana-app-sdk: command not found`

The Go `bin` directory is not in `PATH`. Add it:
```bash
export PATH="$PATH:$(go env GOPATH)/bin"
```

### `generate` fails with CUE errors

CUE is strict about types and constraints. Common causes:
- Undefined field reference
- Type mismatch (`string` where `int` expected)
- Missing import (such as `time` for `time.Time`)

Run `cue vet ./kinds/...` to get more detailed error output.

### `generate` produces no TypeScript output

Check that `codegen.ts.enabled: true` is set in the kind's CUE file and that an output path is configured.

### `go mod tidy` needed after generate

After running `generate`, newly generated Go files may import packages not yet in `go.sum`. Run:
```bash
go mod tidy
```
