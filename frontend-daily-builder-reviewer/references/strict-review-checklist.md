# Strict Review Checklist

## Architecture
- Feature-based folder structure
- Clear separation of UI, state, and data fetching
- Reusable components over copy-paste blocks

## Performance
- Avoid unnecessary rerenders
- Use memoization only where useful
- Handle loading states without layout shifts

## Code Quality
- Clear naming and small components
- Predictable state shape
- Consistent error handling strategy

## Product Readiness
- Empty/loading/error states present
- Responsive behavior verified
- Accessibility basics: focus states and contrast
